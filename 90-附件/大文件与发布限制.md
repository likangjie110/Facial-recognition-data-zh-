## 建议

- 资料总量不大时，可以先直接随仓库发布，保证“文档 + 附件”完整。
- 如果后续工具包版本增多，建议把大型二进制文件迁移到 GitHub Releases 或 Git LFS。
- 发布脚本会提示超过 50 MB 的文件，但不会自动删除、移动或忽略它们。

## 不自动处理的原因

当前目标是生成完整知识库。自动迁移大文件会改变附件访问方式和发布契约，因此保留为显式人工决策。
