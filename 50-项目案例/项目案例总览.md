# 项目案例总览

本区提供四种语言的完整项目调用案例，用于把手册中的照片注册流程落到可运行工程中。

## 协议依据

案例只实现手册中已经明确的公共协议子集：

- 基本帧格式：`SyncWord + MsgID + Size + Data + ParityCheck`。
- `SyncWord` 固定为 `0xEF 0xAA`。
- `ParityCheck` 为去掉 `SyncWord` 后其余字节逐字节 XOR。
- 照片注册命令采用 `0xF7`。
- 首包使用 `Seq=0`，数据包含 `4 byte` 大端照片长度和 `BioType`。
- 后续包使用 `Seq=1..N`，照片数据按每包最多 `246 byte` 分包。

## 工程边界

四个工程都把“协议封包”和“传输通道”分开：

- 协议层负责生成帧、校验、照片分包。
- 默认 `MockTransport` 不连接真实设备，只打印发送帧并模拟成功回复。
- 接入真实模组时，将 transport 替换为串口、USB CDC、JNA SDK 或厂商 DLL 调用即可。

## 推荐验证顺序

1. 先运行 Python 示例，确认协议封包和分包流程。
2. 再运行 Java 或 C++ 示例，确认跨语言生成的帧格式一致。
3. 最后用 Qt 示例接入真实串口，验证现场交互。
