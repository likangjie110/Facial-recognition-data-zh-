# 编辑与发布流程

标签：#发布 #GitHubPages #Docsify #Obsidian

## 本地编辑

1. 用 Obsidian 打开当前文件夹。
2. 从 `README.md` 进入知识库。
3. 新增资料时，优先放在匹配主题目录下建立 Markdown 资料卡。
4. 原始附件保留在根目录，资料卡使用相对链接引用。
5. PDF 有新增或替换后，重新提取全文。
6. 发布前运行检查脚本。

```powershell
.\scripts\extract-pdf-fulltext.ps1
```

```powershell
.\scripts\check-kb.ps1
```

## 本地预览

```powershell
.\scripts\serve-local.ps1
```

脚本会在当前目录启动静态服务，浏览器访问输出的本地地址即可预览 Docsify 站点。

## 首次发布

先在 GitHub 创建一个空仓库，然后执行：

```powershell
.\scripts\publish-github-pages.ps1 -RemoteUrl "https://github.com/likangjie110/Facial-recognition-data-zh-.git"
```

脚本会：

- 如当前目录还不是 Git 仓库，则初始化仓库。
- 配置或更新 `origin` 远程地址。
- 运行知识库基础检查。
- `git add -A`、提交并推送到目标分支。

## GitHub Pages 设置

推荐在 GitHub 仓库中配置：

- Source：Deploy from a branch
- Branch：`main`
- Folder：`/root`

因为 Docsify 站点入口 `index.html` 位于仓库根目录，GitHub Pages 从根目录发布即可。

如果使用仓库内置的 `.github/workflows/pages.yml`，则在 GitHub Pages 设置中把 Source 设为 `GitHub Actions`。发布成功后的访问地址为：

```text
https://likangjie110.github.io/Facial-recognition-data-zh-/
```

## 后续更新

```powershell
.\scripts\publish-github-pages.ps1
```

如果只想本地提交、不推送：

```powershell
.\scripts\publish-github-pages.ps1 -SkipPush
```
